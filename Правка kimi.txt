🟡 Средние проблемы
6. Rate Limiting — только на API, WebSocket без rate limit на handshake
Из CHANGELOG.md:
Rate limiting WebSocket: 5 сообщений/сек
Но в STRUCTURE.md и TECHNICAL.md нет упоминания о rate limit на handshake WebSocket. Злоумышленник может открывать тысячи соединений без аутентификации.
Фикс: Добавить io.use() middleware с IP-based rate limit до handshake:
JavaScript
const connectionAttempts = new Map(); // или Redis
io.use((socket, next) => {
  const ip = socket.handshake.address;
  const attempts = connectionAttempts.get(ip) || 0;
  if (attempts > 10) return next(new Error('Too many connections'));
  connectionAttempts.set(ip, attempts + 1);
  setTimeout(() => connectionAttempts.set(ip, attempts), 60000);
  next();
});
7. BullMQ — setInterval fallback, но нет graceful shutdown для fallback
Из CHANGELOG.md:
Без Redis: setInterval fallback (как было)
Проблема: При fallback на setInterval нет clearInterval в обработчике SIGTERM. Процесс может зависнуть при shutdown.
Фикс: В background.js сохранять intervalId и очищать в shutdown hook.
8. Cache delPattern — Lua-скрипт, но без EVALSHA оптимизации
Из CHANGELOG.md:
Redis invalidate() переписан на Lua-скрипт (атомарное keys + del)
Проблема: Каждый вызов EVAL отправляет весь скрипт по сети. Redis поддерживает EVALSHA — хеш скрипта, который выполняется без передачи тела.
Фикс: Предзагрузить скрипт через SCRIPT LOAD при инициализации:
JavaScript
const sha = await redis.script('LOAD', luaScript);
// затем использовать redis.evalsha(sha, ...)
9. Zod v4 — breaking changes, но package.json уже на v4

Проблема: Zod v4 имеет breaking changes относительно v3 (другой API для .pipe(), .transform(), .refine()). В TECHNICAL.md указано Zod 3.x, а в package.json — v4. Если код писался под v3 — сломается.
Фикс: Проверить все схемы в server/src/utils/validators/ на совместимость с v4. Особенно z.object().pipe() → z.pipeline() в v4.

14. Sentry DSN — опциональный, но без beforeSend фильтра
В TECHNICAL.md:
JavaScript
Sentry.init({ dsn: process.env.SENTRY_DSN })
Проблема: В логи Sentry могут попасть PII (personally identifiable information) — email пользователей, токены, пароли.
Фикс: Добавить beforeSend:
JavaScript
Sentry.init({
  dsn: process.env.SENTRY_DSN,
  beforeSend(event) {
    if (event.exception) {
      // скрыть sensitive data из stack trace
      event.exception.values?.forEach(v => {
        if (v.stacktrace) {
          v.stacktrace.frames?.forEach(f => {
            f.vars = undefined; // убрать локальные переменные
          });
        }
      });
    }
    return event;
  }
});
