Ладно, всё, пишу. Не буду больше копать, а то ответ будет слишком длинным. Сфокусируюсь на 8 проблемах. Нашёл несколько проблем от критичных до мелочей. Вот разбор с конкретными фиксами:

---

## 🔴 Критичные

### 1. Дублирование `QueryClientProvider` — два независимых инстанса кэша
**Где:** `src/main.tsx` + `src/App.tsx`

В `main.tsx` оборачивается в `QueryClientProvider`, но внутри `App.tsx` уже создан свой `QueryClient` и свой `QueryClientProvider`. Мутации и кэш работают в разных мирах.

**Фикс — `src/main.tsx`:**
```tsx
// Убрать QueryClientProvider, оставить только ErrorBoundary
createRoot(document.getElementById('root')!).render(
  <ErrorBoundary>
    <App />
  </ErrorBoundary>,
)
```

---

### 2. `/register` не защищён `ProtectedRoute`
**Где:** `src/App.tsx`

По спеке (AGENTS.md, Этап 6) регистрация должна быть доступна только `admin/super_admin`. Сейчас маршрут `/register` висит наружу без защиты.

**Фикс — `src/App.tsx`:**
```tsx
<Route
  path="/register"
  element={
    <ProtectedRoute adminOnly>
      <Page>
        <Register />
      </Page>
    </ProtectedRoute>
  }
/>
```

---

### 3. `auditLogMiddleware` применяется ДО `authenticateToken`
**Где:** `server/src/app.js`

`mount('/tickets', ticketsRouter, apiLimiter, cacheMiddleware, auditLogMiddleware)` — `auditLog` пытается записать `req.user.id`, но `req.user` появляется только внутри `ticketsRouter` после `authenticateToken`. Логи будут с `userId = undefined`.

**Фикс:** Убрать `auditLogMiddleware` из `mount()` и добавить его внутрь каждого router'а **после** `authenticateToken`. Например, в `server/routes/tickets.js`:

```js
router.use(authenticateToken)
router.use(auditLogMiddleware) // теперь req.user уже есть
```

И в `app.js` упростить `mount`:
```js
function mount(prefix, router, ...mw) {
  app.use(`/api${prefix}`, ...mw, router)
  app.use(`${V1}${prefix}`, ...mw, router)
}

mount('/auth', authRouter, authLimiter)
mount('/tickets', ticketsRouter, apiLimiter, cacheMiddleware) // без auditLogMiddleware
// ... и так для всех
```

---

### 4. Unhandled rejections в `notify.js`
**Где:** `server/src/notify.js`

`createNotification(...)` вызывается без `await` и без `.catch()`. Если Promise упадёт (например, БД недоступна) — процесс крашнется от unhandled rejection.

**Фикс — обернуть все вызовы `createNotification`:**

```js
// В начале файла добавить хелпер
function safeNotify(promise) {
  promise.catch(err => logger.error('Notification failed:', err.message))
}

// И заменить вызовы:
safeNotify(createNotification({ userId: t.creatorId, type: 'ticket_created', title: 'Тикет создан', body: t.title, link: `/tickets/${ticketId}` }))
```

Или, если хотите `await` (блокирующе):
```js
await createNotification({...}).catch(err => logger.error(...))
```

## 🟡 Средние

### 5. Динамический `import` на каждое WebSocket-сообщение
**Где:** `server/src/socket.js`

```js
const { createNotification } = await import('./routes/notifications.js')
```
внутри `socket.on('message:send')` — делает импорт на **каждое** сообщение в чате.

**Фикс:** Вынести импорт наверху файла:
```js
import { createNotification } from './routes/notifications.js'
// или если циклическая зависимость мешает — вынести в отдельный сервисный файл
```

### 6. `warnAdminRedisMissing` — `sendTicketNotification` без `.catch()`
**Где:** `server/src/background.js`

```js
for (const a of admins) {
  if (a.email) {
    sendTicketNotification({...}) // fire-and-forget, нет await/.catch()
  }
}
```

**Фикс:**
```js
sendTicketNotification({ to: a.email, ... }).catch(() => {})
```

---

### 7. `message:delete` в WebSocket — `super_admin` не может удалять
**Где:** `server/src/socket.js`

```js
const isAdmin = socket.userRole === 'admin' || socket.userRole === 'senior_agent'
```

`super_admin` не проходит эту проверку, хотя по иерархии RBAC должен.

**Фикс:**
```js
import { hasRole } from './utils/roleUtils.js'

// внутри обработчика:
const isAdmin = hasRole(socket.userRole, 'senior_agent')
```

## 🟢 Мелочи

### 8. `wsRateLimit` — дробные токены
**Где:** `server/src/socket.js`

```js
entry.tokens = Math.min(WS_LIMIT, entry.tokens + elapsed * (WS_LIMIT / WS_INTERVAL))
```

Может дать `3.7` токенов, и при сравнении `entry.tokens < 1` поведение непредсказуемо.

**Фикс:**
```js
entry.tokens = Math.floor(Math.min(WS_LIMIT, entry.tokens + elapsed * (WS_LIMIT / WS_INTERVAL)))
```
